============================================================
00) ROUTING QUESTIONS (EN/RU)
============================================================
EN — Ask before selecting a method:
- Area & objective: [Controls test / Substantive test] and assertion(s).
- Population: count (N), total book value (TBV), min/max item value, zero/credit items present?
- Risk level (High/Moderate/Low) and reliance on controls (Y/N).
- Tolerable Deviation Rate (TDR) for controls OR Tolerable Misstatement (TM) for substantive.
- Expected Deviation Rate (EDR) OR Expected Misstatement (EM).
- Confidence Level (CL) or Sampling Risk (SR = 1−CL).
- Stratification needed? thresholds by value/date/vendor/SKU.
- MUS suitability? (PPS not appropriate for many negatives/zeros).

RU — Спросить перед выбором метода:
- Раздел и цель: [Тест контроля / Тест по существу] и утверждения.
- Генеральная совокупность: количество (N), суммарная балансовая стоимость (TBV), мин/макс, есть нулевые/кредитовые позиции?
- Уровень риска (Высокий/Средний/Низкий) и опора на контроль (Да/Нет).
- Допустимая частота отклонений (TDR) для контролей ИЛИ Допустимое искажение (TM) для процедур по существу.
- Ожидаемая частота отклонений (EDR) ИЛИ Ожидаемое искажение (EM).
- Уровень доверия (CL) или риск выборки (SR = 1−CL).
- Нужна ли стратификация? пороги по сумме/датам/поставщикам/SKU.
- Применимость MUS? (PPS не подходит при множестве кредитов/нулей).

============================================================
01) METHOD SELECTION MATRIX (EN/RU)
============================================================
EN — Choose method based on objective & population:
- Attribute sampling (controls) → binary deviations; target: TDR vs EDR; output: deviation rate & upper deviation rate.
- MUS / PPS (monetary unit sampling, substantive) → best for overstatement risk in assets/AR/revenue.
- Classical Variables (Mean-per-unit, Ratio, Difference) → mixed/bi-directional risk, many negatives/zeros, inventories.
- Haphazard is NOT acceptable; use random or systematic selection with random start.

RU — Выбор метода:
- Атрибутная выборка (контроли) → бинарные отклонения; цель: TDR vs EDR; вывод: частота отклонений и верхняя граница.
- MUS / PPS (денежно‑единичная выборка) → лучше для риска завышения активов/ДЗ/выручки.
- Классическая переменная (средняя на единицу, долевая, разностная) → двусторонний риск, много кредитов/нулей, запасы.
- Случайный/систематический отбор с случайным стартом; «произвольный» не допускается.

============================================================
02) ATTRIBUTE SAMPLING — CONTROLS TESTS (EN/RU)
============================================================
EN — Inputs: N, TDR, EDR, CL (or SR), risk level.
Sample Size: use binomial/Poisson or firm factor tables (preferred). If unavailable, request confirmation to use conservative factors.
Selection: random or systematic with random start.
Evaluation: compute sample deviation rate (SDR), project to population, compute Upper Deviation Rate (UDR).
Decision: if UDR ≤ TDR → control effective (for scope purposes); else ↑ scope or redesign.

RU — Входы: N, TDR, EDR, CL (или SR), уровень риска.
Размер выборки: по таблицам факторов фирмы (предпочтительно) или биномиальным приближениям. При отсутствии — запросить подтверждение по консервативным факторам.
Отбор: случайный или систематический со случайным стартом.
Оценка: SDR, проекция, верхняя граница UDR.
Вывод: если UDR ≤ TDR → контроль приемлем; иначе расширить объем или изменить подход.

03) MUS / PPS — SUBSTANTIVE (EN/RU)
============================================================
EN — Inputs: TBV, TM, EM, CL (or SR), Risk level. Remove known misstatements from population if appropriate.
Key Concepts:
- Sampling Interval (SI) and Sample Size (n): n = TBV / SI; SI = TM / Reliability Factor (RF).
- Random Start: choose a number U(1, SI). Systematic selection: cumulative BV crosses RS + k×SI.
- Tainting factor = (Overstatement) / (Book Value). Projected misstatement = Tainting × SI (basic method).
- Upper Misstatement Limit (UML) = Projected + Increment for allowance (per RF table).

RU — Входы: TBV, TM, EM, CL (или SR), уровень риска. При необходимости исключить известные искажения.
Ключевые понятия:
- Интервал (SI) и размер n: n = TBV / SI; SI = TM / Коэффициент надежности (RF).
- Случайный старт: число U(1, SI). Систематический отбор: кумулятив BV пересекает RS + k×SI.
- Коэффициент «загрязнения» (tainting) = (Завышение) / (Балансовая стоимость). Проекция = tainting × SI.
- Верхняя граница искажения (UML) = Проекция + надбавка (по RF).

04) CLASSICAL VARIABLES — SUBSTANTIVE (EN/RU)
============================================================
EN — When to use: many negatives/zeros, bidirectional risk, inventories or pricing tests.
Methods: Mean-per-unit (MPU), Ratio, Difference.
Inputs: sample size via firm table or formulas using std. deviation/variability; CL; TM.
Evaluation: compute point estimate & precision; derive Upper Misstatement bound; compare to TM.
RU — Когда применять: много кредитов/нулей, двусторонний риск, запасы/ценообразование.
Методы: MPU, Долевая, Разностная. Входы: размер выборки по дисперсии/вариабельности, CL, TM. Оценка: точечная оценка и погрешность; верхняя граница; сравнение с TM.

============================================================
05) STRATIFICATION & TOP-SAMPLING (EN/RU)
============================================================
EN — Rules:
- Always consider top-sampling of high-value/critical items (100%).
- Stratify by value to reduce variability (large, medium, small). Define cut-offs.
- Separate unusual/related-party items into targeted testing.
RU — Правила: проверка 100% крупных/критичных; стратификация по сумме (крупные/средние/мелкие); выделение нетипичных/связанных сторон на адресные процедуры.

============================================================
06) DOCUMENTATION & REPRODUCIBILITY (EN/RU)
============================================================
EN — Always store: seed, random start, selection list, rationale for method & parameters, tables used, calculation sheet, and conclusion reference to ISA 530/330.
RU — Хранить: seed, случайный старт, список позиций, обоснование метода/параметров, таблицы, расчетный лист, и вывод с ссылками МСА 530/330.

5) Conclusion with ISA 530 reference and scope impact.
